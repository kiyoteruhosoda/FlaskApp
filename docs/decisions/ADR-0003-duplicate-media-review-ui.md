# ADR-0003: 重複メディアは「人手レビュー画面」で削除する

- ステータス: Accepted
- 日付: 2026-06-28

## コンテキスト

Google Photos / ローカル取り込みで同一・近似のメディアが重複登録されることがある。
画像ハッシュ（`hash_sha256` 完全一致 / `phash` 近似一致）で自動削除する案もあったが、
誤検出で別物を消すリスクがある。

## 決定

自動削除はせず、**重複候補をグループ化して提示し、人が残す1枚を選んで他を削除する
レビュー画面**を提供する。削除は**ソフト削除（`is_deleted=True`）**で、originals は
ストレージ（NAS）に残す。

- バックエンド: `GET /api/media/duplicates`（`media:view`）が重複グループを返す。
  - `exact` = `hash_sha256` 一致、`similar` = `phash` 一致（exact のメンバーは similar から除外）。
  - 削除は既存 `POST /api/media/bulk-actions`（`media:delete`）を再利用。
- フロント: `/media/duplicates` 画面。グループごとにサムネイル比較→残す1枚を選択→他を削除。

## 選択肢と理由

- 案A（採用）: 人手レビュー UI。誤削除リスクを人の確認で低減。ソフト削除＋原本保持で
  復元可能。`exact`/`similar` を分けて確信度を提示。
- 案B: ハッシュ一致で自動削除。phash は近似一致のため誤検出があり、不可逆操作と相性が悪い。
- 案C: 取り込み時の重複防止のみ（UI なし）。過去に混入した重複を掃除できない。

## 影響

- GP は ADR 連携の `google_media_id` 一意化（復活方式）で新規重複の混入が止まるため、
  本 UI は主に過去分の掃除用途。
- 物理削除の確定（ファイル除去）は本 UI のスコープ外。必要なら別バッチで is_deleted を
  対象に行う。
